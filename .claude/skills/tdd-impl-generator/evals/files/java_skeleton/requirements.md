# 用户管理模块需求文档

## FR-001 用户注册
- 用户名长度必须在 2~50 个字符之间
- 邮箱必须符合 RFC 5322 格式
- 密码长度 ≥ 8 位，且必须包含至少一个大写字母、一个小写字母、一个数字
- 注册成功后系统发送欢迎邮件到注册邮箱

## FR-002 邮件发送策略
- 欢迎邮件属于非关键路径：邮件发送失败不影响注册成功
- 邮件发送异步执行，不阻塞注册响应
- 邮件失败时记录 WARN 级别日志，包含 userId 和失败原因

## FR-003 邮箱唯一性
- 邮箱地址在系统中全局唯一，不区分大小写（alice@example.com 与 ALICE@EXAMPLE.COM 视为同一邮箱）
- 注册时发现邮箱已存在，返回明确的业务错误（HTTP 409），错误消息中包含"already registered"

## FR-004 用户查询
- 支持按用户 ID 查询
- 查询返回的 DTO 不包含密码哈希字段
- 用户不存在时返回空（不抛出异常），由调用方决定处理方式

## FR-005 用户信息更新
- 用户名（name）可以更新
- 邮箱地址一经注册不可更改（不可变字段）
- 尝试更改邮箱时返回明确错误（HTTP 422），错误消息中包含"immutable"
- 更新的字段值须通过与注册相同的格式校验

## FR-006 账户注销
- 注销采用软删除策略：状态置为 DEACTIVATED，不物理删除数据
- DEACTIVATED 状态用户不可登录（由认证层控制，本服务只负责状态变更）
- 注销操作不可逆（本版本不支持账户恢复）

## FR-007 安全要求（来自 SEC 文档）
- 密码必须使用 bcrypt 加密存储，不得明文落库
- 任何日志语句中不得出现密码明文或密码哈希值
- 邮箱在日志中可以显示（属于非敏感信息）
